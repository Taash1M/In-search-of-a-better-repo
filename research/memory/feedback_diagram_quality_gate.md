Every diagram generated by azure_diagrams must be validated BEFORE embedding into a document or presentation. Never insert a diagram without checking it first.

**Checklist (all must pass):**
1. **Missing icons** — verify every node rendered with its intended icon (not a fallback colored box with initials). Call `load_icon(key)` to pre-check, or inspect the PNG for fallback boxes.
2. **Text overlap** — labels and sublabels must not overlap adjacent node labels, connection labels, or boundary labels. Keep labels ≤15 chars; use sublabels for detail.
3. **Distortion / squashing** — aspect ratio must be preserved. Icons should appear square, not stretched. Boundaries should have reasonable proportions.
4. **Arrow/icon overlap** — arrows should not pass through icons or labels. Use `shrinkA`/`shrinkB` ≥ 30 and `curve` offsets for close parallel arrows.
5. **Node spacing** — minimum 2.6 units between horizontally adjacent nodes (flow diagrams), 3.0+ for pptx preset. Vertical flows need 2.6+ between stages.
6. **Unique labels** — every node label within a single diagram must be unique. Use qualifiers (e.g., "Bronze Layer" vs "Silver Layer", not both "Databricks").

**Why:** The Customer MDM v1 diagrams had ADF icon missing (not in registry), text overlapping between adjacent nodes, and cramped layouts. User had to request a revision round — this should be caught automatically.

**How to apply:** After generating any diagram PNG, open/read the image to visually inspect it. If any check fails, adjust node positions/spacing/labels and regenerate. Repeat until all checks pass. Only then embed into the target document. This applies to all skills that call azure_diagrams: docx-beautify, powerpoint-create, ai-ucb-docs.
