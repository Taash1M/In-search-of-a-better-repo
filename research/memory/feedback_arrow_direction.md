---
name: Arrow direction convention in docs
description: When linking two sections with arrows, each arrow points TOWARD the other section — not both in the same direction.
type: feedback
---

When two sections reference each other with arrows (e.g., a step card pointing to a credentials box), the arrows should point TOWARD the other section:

- The **source section** (Step 3) gets a ⬇️ arrow + "see below" — pointing DOWN to the credentials
- The **target section** (credentials box) gets a ⬆️ arrow — pointing UP back to the step

**Why:** The user corrected this twice. Both arrows pointing down was confusing — the credentials box arrow should guide the eye back up to where you need the credentials (Step 3).

**How to apply:** Whenever placing directional indicators between two doc sections, make each arrow point toward the *other* section, creating a visual round-trip.
